## 1. وصف المشروع

**يمينك** هو نظام رعاية صحية متكامل يهدف لمتابعة الحالة الصحية لكبار السن عن بُعد. يتضمن:
- جمع قراءات صحية (سكر، ضغط، حرارة، أكسجين)
- تحليل القراءات بالذكاء الاصطناعي (Claude API)
- تنبيهات فورية للأسرة عبر واتساب
- شات صحي تفاعلي مدعوم بالذكاء الاصطناعي
- سوق خدمات (تمريض، نقل طبي، طبيب منزلي)

## 4. تقييم المخاطر

### 4.1 مخاطر عالية

| المخاطرة | الاحتمال | الأثر | الإجراء الوقائي |
|---|---|---|---|
| تسرب بيانات المسن الصحية | متوسط | **عالي** | RLS في Supabase + تشفير AES-256 at rest |
| إرسال PII لـ Claude API | عالي | **عالي** | ✅ تم: `anonymizer.util.ts` يزيل الاسم والهوية |
| وصول غير مصرح به للبيانات | متوسط | عالي | ✅ تم: `audit_logs` + JWT authentication |
| عدم قدرة المسن على سحب الموافقة | منخفض | متوسط | ✅ تم: شاشة إعدادات الخصوصية + سحب فوري |

### 4.2 مخاطر متوسطة

| المخاطرة | الاحتمال | الأثر | الإجراء الوقائي |
|---|---|---|---|
| فقدان البيانات عند حذف الحساب | منخفض | متوسط | حذف فوري + سجل في `data_deletion_requests` |
| تقصير في التنبيهات الصحية | منخفض | متوسط | نظام Quiet Hours + Redis queue |
| وصول الممرض لبيانات غير ضرورية | متوسط | متوسط | RBAC مع RLS policies |

## 5. الإجراءات الوقائية المطبقة

### 5.1 التشفير
- **At rest:** AES-256 عبر Supabase (PostgreSQL)
- **In transit:** TLS 1.3 لجميع اتصالات API
- **Tokens:** تخزين JWT في `FlutterSecureStorage` (Keychain/Keystore)

### 5.2 إخفاء الهوية
- `anonymizer.util.ts`: يستبدل الاسم بـ "المريض"، العمر بنطاق عمري، ويحذف الهوية الوطنية ورقم الجوال
- `pseudonymizeId()`: HMAC-SHA256 للمعرفات عند الحاجة للربط
- `buildPatientContext()`: يطلب فقط الحقول الطبية من DB (name/dosage/frequency)

### 5.3 إدارة الموافقة
- شاشة موافقة إلزامية بعد أول تسجيل دخول
- Privacy as Default: الخيارات الاختيارية معطلة تلقائياً
- سحب الموافقة متاح في أي وقت من إعدادات الخصوصية
- تسجيل كل منح/سحب في `audit_logs`

### 5.4 حقوق أصحاب البيانات
- **تحميل البيانات:** `GET /privacy/my-data` — JSON كامل
- **حذف الحساب:** `DELETE /privacy/account` — مسح شامل مع CASCADE
- **سجل الوصول:** `GET /privacy/audit-log` — من وصل ومتى

### 5.5 أمان البنية التحتية
- استضافة AWS Bahrain (me-south-1) — إقامة البيانات داخل المنطقة
- Row-Level Security على جميع الجداول الحساسة
- Rate Limiting في API Gateway
- Validation Pipe مع whitelist في NestJS

## 6. خطة الاستجابة لحوادث تسرب البيانات

1. **اكتشاف** → مراقبة `audit_logs` للأنماط المشبوهة
2. **احتواء** → تعليق الحسابات المتأثرة فوراً
3. **إبلاغ** → إخطار الجهة المختصة (سدايا) خلال 72 ساعة
4. **إخطار** → إبلاغ أصحاب البيانات المتأثرين
5. **معالجة** → تحقيق وإصلاح الثغرة
6. **توثيق** → تحديث DPIA وإجراءات الأمان
